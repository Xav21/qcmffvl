Version Web
La version web a été conçue pour être multi-support (mobile / tablette / desktop) et multi-OS (Linux / Windows / MAC...).

La totalité du questionnaire n'est chargé qu'une seule fois d'un point de vue réseau.
Cela veut dire qu'il vous suffit de charger la page sur votre mobile avant de partir, et vous pourrez vous faire d'innombrables QCMs FFVL, sans avoir jamais besoin de connexion data !

La version web a été testée comme fonctionnant correctement avec Chrome et Firefox.
Merci d'utiliser un de ces 2 navigateurs si vous expérimentez des soucis.
Application Android
L'application Android fait un usage intelligent du réseau. Au lancement de l'app Android, celle-ci regarde si l'appareil a de la data activée et une connection internet valide, auquel cas elle télécharge la dernière version web (et tout le questionnaire), sinon elle utilise la dernière version téléchargée.
Cela assure une utilisation hors ligne facilitée et une mise à jour automatique.

Note : Seul le mode révision est disponible depuis l'appli Android pour le moment.
CHANGELOG
Version 3.1
Améliorations / Changements
  • Création de l'application Android, pour une meilleure expérience offline.
Version 3.0
Améliorations / Changements
  • Ajout du questionnaire Delta.
  • Possibilité de signaler un problème avec une question en cliquant sur l'encart de la question.
  • Nouvelle répartition des questions suivant un pourcentage par catégorie.
  • Ordre aléatoire des réponses pour chaque question. L'ordre de tout le questionnaire (questions + réponses) reste cependant le même pour un QCM ID donné.
  • Le QCM ID (bouton partage) est maintenant lié à la version de QCM, les anciens QCM IDs ne sont plus compatibles avec les versions 3.X (ou versions supérieures), qui intègrent cette fonctionnalité.
  • Retour à un navbar collapse sur 1200px.
Bugfixes
  • Correction d'une mauvaise génération de l'URL de partage où l'ID pouvait être répété 2 fois.
  • Modification du CSS impression pour éviter un effacement aléatoire de la barre entre la question et les réponses.
Version 2.2
Améliorations / Changements
  • Simplification du code couleur : si une bonne réponse n'est pas cochée, alors l'afficher en gras (au lieu de orange/rouge).
  • Ajout d'une modale pour partager le QCM ID, et permettre le chargement d'un questionnaire donné via une URL.
Version 2.1
Améliorations / Changements
  • Suppression des boutons corriger / résultat en mode examen.
  • Retour au début de la page quand le mode de questionnaire est modifié.
Version 2.0
Améliorations / Changements
  • Nouveau code couleur de correction plus intuitif (vert, orange, rouge, en fonction des réponses utilisateur)
  • Ajout du QCM ID, qui identifie de manière unique un questionnaire, une catégorie, un niveau et un nombre de questions. Intègre un checksum pour éviter les erreurs dans les partages d'ID.
  • Ajout du mode examen papier (candidat / examinateur), avec CSS pour l'impression. Les questionnaires sont identifiables par leur "code" CRC.
  • Affichage des boutons corriger et recharger quand la navbar est fermée pour les displays mobiles/tablettes.
  • Page "A propos" : Ajout d'un bouton "Retour QCM" plus intuitif que de cliquer sur "QCM FFVL".
  • Réduction de l'opacité du contenu quand la navbar est ouverte.
Bugfixes
  • Affichage sur la même ligne des réponses et des points (après correction) sur les mobiles.
  • Collapse de la navbar à 1200px pour éviter de manger la 1ere question sur toutes les tailles (retravail de la navbar pour rentrer dans 1200 px).
  • Checkboxs complètement désactivées quand la navbar est ouverte.
Version 1.2
Bugfixes
  • Changer les options en étant dans la page "à propos" redirige vers la page QCM au lieu d'afficher "Chargement..." au milieu de la page.
Version 1.1
Améliorations / Changements
  • En mode smartphone/tablette, quand le formulaire est cliqué, la barre de menu se réduit (et rien ne se coche/décoche)
